## 技術スタック

- フロント、Next.js 16（App Router、Turbopack）+ React 19 + TypeScript
- バックエンド、Supabase（Auth / Postgres / Storage / Realtime / RPC）、すべて RLS で多層防御
- 決済、Stripe（payment mode、Webhook 署名検証あり）
- メール、Resend
- ホスティング、Vercel + Cloudflare（CDN + WAF + Bot Fight）
- 解析、Microsoft Clarity + Google Analytics 4（IP 匿名化）

## 診断対象範囲

### Web アプリケーション層

- 認証フロー（メール magic link + Google OAuth）
- 1968 認証フロー（誓約 + 自由記述）
- 会員管理（一般会員 / 認証済 / 創設メンバー / 応援団）
- 掲示板（カテゴリ閲覧・投稿、画像/動画添付）
- ダイレクトメッセージ（メディア添付あり、運営宛例外あり）
- 応援団 Stripe 決済フロー（payment_intent + Webhook）
- 管理画面（admins ロール、CRUD 一式）
- ベータ応募フォーム（公開エンドポイント、honeypot あり）

### API 層

- `/api/stripe/checkout` `/api/stripe/webhook` `/api/stripe/portal`
- `/api/views`
- `/auth/*`、Supabase 連携
- Server Actions 約 30 種（profile、verification、messages、admin など）

### データベース・ストレージ

- Supabase Postgres（全テーブルに RLS）
- Storage バケット、`profile-avatars` `post-media`
- 旧 `verification-documents`（現在は新規利用なし、過去データ閲覧のみ）

### 想定される脅威モデル

- 50 代向けサービス → ソーシャルエンジニアリング、フィッシングへの耐性
- 同年代詐称（1968 年生まれを偽る） → 認証フローの突破
- なりすまし、複数アカウント
- 不正な投稿スクレイピング、商業利用
- DM 経由の嫌がらせ、勧誘
- アバター・投稿画像の悪用
- Stripe 決済の二重請求、改竄

## 診断項目（最低限）

OWASP Top 10（2021）+ ASVS L2 を基準に、

1. インジェクション（SQL、NoSQL、Command、LDAP、OS）
2. 認証・セッション管理（magic link 偽装、トークン漏洩）
3. アクセス制御（RLS バイパス、IDOR、admin 権限昇格）
4. セキュリティ設定ミス（HTTP ヘッダー、CSP、CORS）
5. 安全でないデザイン（ビジネスロジックの破壊）
6. 脆弱なコンポーネント（依存ライブラリ）
7. ID & 認証の障害
8. ソフトウェアとデータの完全性（CSRF、デシリアライゼーション）
9. ログ監視の不足
10. SSRF
11. ファイルアップロード（画像差し替え、xss in svg）
12. 決済フローの整合性
13. レート制限、Bot 対策の有効性
14. プライバシーポリシー遵守状況
